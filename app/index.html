inputValue.CharCodeAt()

This function has the ability to take in any known character. It works by converting each character into it's unicode equivalent up to value 65536 (0x65536). The values from 65536 (0x65536) to 1114111 (0x10FFFF) are unavailable using this method.
From MDN: "This is because the higher code points are represented by a pair of (lower valued) "surrogate" pseudo-characters which are used to comprise the real character."
